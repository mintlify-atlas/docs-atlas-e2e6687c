# Key features

Biovity provides a comprehensive set of features designed specifically for the scientific job market. Here are the 8 core capabilities that make the platform valuable for both job seekers and employers.

## 1. Specialized job search

Advanced search functionality tailored to scientific careers in biotechnology, biochemistry, chemistry, and health sectors.

<CardGroup cols={2}>
  <Card title="Scientific categories" icon="flask">
    Browse jobs organized by scientific discipline, research areas, and technical specializations
  </Card>
  
  <Card title="Advanced filters" icon="filter">
    Filter by location, salary range, experience level, contract type, and scientific field
  </Card>
</CardGroup>

### Key capabilities

- **Multi-field search**: Search by job title, company, location, or keywords
- **Category filtering**: Biotechnology, biochemistry, chemistry, chemical engineering, health, and more
- **Salary range filters**: View jobs within specific Chilean peso (CLP) salary ranges
- **Location-based search**: Filter by Chilean cities and regions
- **Experience levels**: Entry-level, mid-level, senior, and principal positions
- **Contract types**: Full-time, part-time, internships, temporary, and contract work

<Tip>
  The search interface is optimized for scientists who know exactly what they're looking for while also supporting exploratory browsing.
</Tip>

### Technical implementation

Located in `app/dashboard/employee/search/` with:
- Server-side filtering for performance
- Real-time search results
- Persistent search preferences
- Mobile-responsive design with TailwindCSS

## 3. Better Auth authentication system

Secure, flexible authentication powered by Better Auth with custom user types and fields.

### Authentication features

<CardGroup cols={2}>
  <Card title="Email/password" icon="envelope">
    Traditional authentication with secure password hashing and validation
  </Card>
  
  <Card title="User types" icon="users">
    Separate flows for employees (job seekers) and organizations (employers)
  </Card>
  
  <Card title="Session management" icon="clock">
    7-day sessions with automatic renewal and secure logout
  </Card>
  
  <Card title="Rate limiting" icon="shield">
    10 requests/minute per IP to prevent abuse and brute force attacks
  </Card>
</CardGroup>

### Custom user fields

Better Auth is configured with custom fields in `lib/auth.ts`:

```typescript
// Custom user fields
type: 'employee' | 'organization'
profession: string           // Scientific field/role
avatar: string              // Profile image URL
isActive: boolean           // Account status
organizationId: string      // Link to parent org (for employees)
```

### Authentication architecture

- **Server config**: `lib/auth.ts` - Better Auth server instance
- **Client config**: `lib/auth-client.ts` - Client-side auth hooks
- **API endpoint**: `app/api/auth/[...all]/route.ts` - Handles all auth requests
- **Protected routes**: Middleware checks authentication for dashboard access

<Info>
  The dual user type system (employee/organization) enables different registration flows, dashboards, and permissions for job seekers vs. employers.
</Info>

## 4. Application tracking and management

Comprehensive system for managing job applications throughout the hiring process.

### For job seekers

Located in `app/dashboard/employee/my-applications/`:

- **Application history**: View all submitted applications with timestamps
- **Status tracking**: Monitor application progress (pending, reviewed, interview, rejected, accepted)
- **Application details**: Access original job posting and your submitted materials
- **Notifications**: Receive updates when application status changes
- **Withdrawal option**: Cancel applications if circumstances change

### For employers

Located in `app/dashboard/employee/applications/` (employer view):

- **Candidate pipeline**: Organize applications by stage in the hiring process
- **Bulk actions**: Update multiple application statuses simultaneously
- **Candidate comparison**: Side-by-side review of applicant qualifications
- **Communication**: Message candidates directly through the platform
- **Filtering**: Sort and filter applications by date, status, qualifications

<Tip>
  Application data is stored in PostgreSQL with proper indexing for fast queries even with thousands of applications.
</Tip>

## 7. Analytics and metrics

Data-driven insights for both job seekers and employers to optimize their use of the platform.

### Employee metrics

Located in `app/dashboard/employee/metrics/`:

- **Application analytics**: Number of applications, response rates, success rates
- **Profile views**: Track how often employers view your profile
- **Search appearances**: See how often your profile appears in employer searches
- **Engagement metrics**: Measure effectiveness of your job search strategy
- **Time-to-hire**: Average time from application to offer
- **Salary insights**: Compare your expectations to market rates

### Organization metrics

Employers track:

- **Job posting performance**: Views, applications, and applicant quality per posting
- **Time-to-fill**: Average days to fill open positions
- **Candidate pipeline**: Applicants at each stage of the hiring funnel
- **Source tracking**: Where top candidates discover your jobs
- **Hiring trends**: Historical data on hiring patterns and seasonality

<Info>
  Metrics are calculated using utility functions from `lib/utils.ts` with proper date formatting and Chilean peso (CLP) currency handling.
</Info>

### Visualization

- Charts powered by React components
- Date range selectors for custom reporting periods
- Export capabilities for further analysis
- Mobile-responsive dashboard layouts

## Feature integration

These 8 features work together to create a cohesive platform experience:

1. **Search** helps discover opportunities
2. **Dashboard** provides centralized access to all features
3. **Authentication** secures user data and enables personalization
4. **Application tracking** manages the hiring process
5. **Profiles** showcase qualifications and preferences
6. **Messaging** facilitates communication
7. **Analytics** provide insights for optimization
8. **Calendar** keeps everyone organized and on schedule

<Note>
  All features are built with TypeScript for type safety, TailwindCSS for responsive design, and PostgreSQL for reliable data storage.
</Note>

## Technical architecture

Features are organized using Next.js App Router conventions:

- **Route-based organization**: Each feature has dedicated routes in `app/`
- **Component modularity**: Shared UI components in `components/ui/`
- **Type safety**: TypeScript interfaces in `lib/types/` define data structures
- **Utility functions**: Common operations in `lib/utils.ts` (dates, currency, badges)
- **Database schema**: PostgreSQL tables with migrations in `lib/db/migrations/`
